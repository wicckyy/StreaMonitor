## 2025-05-04 - [Missing Authentication on Sensitive Endpoints]
**Vulnerability:** The `/video/<user>/<site>/<path:filename>` and `/clear` endpoints in `streamonitor/managers/httpmanager/httpmanager.py` were missing the `@login_required` decorator, allowing unauthenticated access to potentially sensitive recorded videos and allowing unauthenticated users to trigger state-altering actions like clearing modals.
**Learning:** In Flask applications utilizing custom decorators like `@login_required` for basic authentication, it is easy to forget to apply the decorator to new or less obvious endpoints, especially when routes are manually defined with `@app.route`. The lack of a centralized authentication middleware means each route must be individually secured.
**Prevention:** Ensure that every route definition that is not explicitly intended to be public is reviewed for the presence of the `@login_required` decorator. Consider implementing a more centralized or declarative authentication enforcement mechanism if the number of routes grows.
